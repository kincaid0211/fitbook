# 非书 UI 视觉稿

更新日期：2026-05-12

本页记录「非书」Demo 阶段的关键页面 UI 视觉方向。视觉稿基于已确认的产品设定生成：

- 产品名：非书
- Slogan：读出自己的书。
- 副标题：从一个知乎问题出发，在启发式阅读和主动探索中生成你的知识读本。
- 视觉基调：知乎蓝白、清晰可信、简洁出版感、内容产品感。

## 1. 落地页

落地页优先服务黑客松 Demo：先讲清产品意义，再引导用户开始第一本非书。

![非书落地页 UI 视觉稿](../assets/ui/feishu-landing-ui.png)

## 2. 我的非书

「我的非书」需要覆盖冷启动场景：用户还没有自己的非书时，先展示明确的开始入口，并用示例路线说明产品最终形态。

![我的非书 UI 视觉稿](../assets/ui/feishu-library-ui.png)

## 3. 非书详情页

详情页展示用户完成探索后生成的个人知识读本，包括封面、目录、作者贡献和阅读路线。

![非书详情页 UI 视觉稿](../assets/ui/feishu-detail-ui.png)

## 设计要点

- 页面文案以用户为中心，强调“你读出的路线”和“你的知识读本”。
- 示例内容用于解决冷启动，不伪装成用户真实生成内容。
- 每本非书应有统一风格的插画封面，并根据主题呈现差异。
- 作者信息是核心内容结构的一部分，需在卡片和详情页中保留。
- 分享功能在 Demo 中可以作为可见的未来能力入口。
